== Context and Scope

In diesem Kapitel wird das System gegenüber externen Akteuren abgegrenzt und der typische Ablauf der Nutzung beschrieben.

=== Business Context

Der Business Context beschreibt den Ablauf aus Sicht der Benutzer (Studierende / Robotik-Interessierte).
Das folgende Diagramm zeigt eine typische Nutzung des Systems.

[plantuml, target="diagram-bpmn-process", format="png"]
----
@startuml
|Benutzer|
start
:Definiere Test-Szenario;
:Schreibe Python-Steuerlogik;
:Wähle Betriebsmodus\n(virtual / real / both);

|#LightBlue|Python API|
:Starte Skript;
:Validiere Eingaben;

if (Simulation aktiv?) then (ja)
  :Sende UDP (5005/5006);
  |#LightGreen|Unity Simulation|
  :Empfange Daten;
  :Simuliere Bewegung;
  :Visualisiere Ergebnis;
else (nein)
endif

if (Hardware aktiv?) then (ja)
  |#LightBlue|Python API|
  :Sende Befehle über Serial;
  |#White|Roboter|
  :Führe Bewegung aus;
else (nein)
endif

|Benutzer|
:Analyse der Ergebnisse;
stop
@enduml
----

[cols="1,3", options="header"]
|===
| Akteur | Verantwortung im Prozess

| **Benutzer**
| Erstellt die Programmlogik und bewertet das Verhalten des Roboters in der Simulation oder auf der Hardware.

| **Python API**
| Dient als zentrale Schnittstelle (Fassade) zum System.
Sie vereinfacht die Nutzung, prüft Eingaben und verteilt Befehle je nach Betriebsmodus an Simulation und/oder Hardware.

| **Unity Simulation**
| Stellt die Bewegung des Roboters visuell dar und ermöglicht das Testen von Szenarien.

| **Hardware**
| Führt die Befehle auf dem realen Roboter aus.

|===

Der technische Kontext zeigt die Systemgrenzen sowie die Kommunikationswege zwischen den einzelnen Komponenten.

==== Externe Schnittstellen

[cols="1,1,2", options="header"]
|===
| Schnittstelle | Protokoll | Zweck

| **Benutzer-API**
| Lokal (Python)
| Stellt Funktionen wie `send_angles()`, `set_color()` oder `spawn()` zur Verfügung.

| **Hardware-Kanal**
| Serial (115200 Baud)
| Überträgt Steuerbefehle an den realen Roboter.

| **Simulations-Kanal**
| UDP (JSON)
| Überträgt Bewegungs- und Zustandsbefehle an die Simulation (Port 5005).

| **Welt-Kanal**
| UDP (JSON)
| Überträgt Befehle zur Verwaltung der Simulationsumgebung (Port 5006).

| **Trennung der Kanäle**
| -
| Die Kommunikation zur Simulation ist in zwei Kanäle aufgeteilt.
Der Roboterkanal ist für zeitkritische Bewegungsbefehle zuständig, während der Weltkanal für weniger häufige Aktionen wie das Erstellen oder Löschen von Objekten verwendet wird.

|===

==== Beispielhafte JSON-Nachrichten

Die Kommunikation zwischen Python und Unity erfolgt über strukturierte JSON-Nachrichten.
Dabei wird zwischen Roboterbefehlen und Weltbefehlen unterschieden. Beide Nachrichtentypen
verwenden eine gemeinsame Grundidee: Jede Nachricht enthält eine Protokollversion `v` sowie
ein Feld `type`, das den Befehl eindeutig beschreibt.

Die konkreten Nachrichten werden auf Python-Seite durch das Modul `protocol` erzeugt.
Dadurch bleibt der Aufbau der übertragenen Daten zentral definiert und konsistent.

.Beispiel einer Nachricht für eine asynchrone Gelenkbewegung
[source,json]
----
{
  "v": 1,
  "type": "move_joints",
  "j1": 0.0,
  "j2": 20.0,
  "j3": -15.0,
  "j4": 10.0,
  "speed": 40
}
----

Diese Nachricht wird über den Roboterkanal an die Simulation übertragen und beschreibt den
Soll-Zustand der vier Gelenke sowie die gewünschte Bewegungsgeschwindigkeit.

.Beispiel einer Nachricht für die Werkzeugwahl
[source,json]
----
{
  "v": 1,
  "type": "set_end_effector",
  "tool": "gripper"
}
----

Diese Nachricht wählt das aktive Werkzeug des Roboters. Unterstützt werden insbesondere
`gripper` und `pump`.

.Beispiel einer Nachricht für das Erzeugen eines Objekts in der Simulationswelt
[source,json]
----
{
  "v": 1,
  "type": "spawn",
  "object_type": "box",
  "name": "box_1",
  "position": {
    "x": 0.2,
    "y": 0.0,
    "z": 0.1
  },
  "rotation": {
    "x": 0.0,
    "y": 0.0,
    "z": 0.0
  },
  "scale": {
    "x": 0.05,
    "y": 0.05,
    "z": 0.05
  },
  "color": {
    "r": 255,
    "g": 0,
    "b": 0
  }
}
----

Diese Nachricht wird über den Weltkanal übertragen und beschreibt ein neues Objekt, das in
Unity erzeugt werden soll. Neben Typ und Name enthält sie auch Transformations- und
Darstellungsparameter wie Position, Rotation, Skalierung und Farbe.

Die Beispiele zeigen bewusst nur einen kleinen Ausschnitt des Protokolls. Weitere Befehle
wie `sync_move_joints`, `led`, `set_gripper_state`, `pump`, `delete` und `clear` folgen
demselben Grundprinzip.
