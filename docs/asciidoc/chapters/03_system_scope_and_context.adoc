==== Akteuere und Interaktionen:

|===
| Akteur | Beschreibung | Interaktion mit dem System

| *Benutzer / Entwickler*
| Interagiert mit dem System über die bereitgestellte Python API. Dabei beinhaltet diese API alle wichtigen Funktionen, um den Roboterarm zu steuern. Zudem sind auch Funktionen beinhaltet, welche es ermöglichen, die Simulationsumgebung zu konfigurieren, um so verschiedene Szenarien mit verschienen Objekten zu erstellen. Dabei besistzt die API bereits 3 standart Szenarien, welche es ermöglichen, die Simulation schnell zu testen und zu nutzen. Es können aber auch eigene Szenarien erstellt werden, um so die Möglichkeiten der Simulation voll auszuschöpfen.
| - Roboterarm steuern

- Szenarien erstellen und nutzen
|===

Dabei ist in diesem Projekt der Benutzer gleichzeitig auch der Entwickler, da die Simulation hauptsächlich für Studenten und Fachleute entwickelt wurde, welche den Roboterarm in einer virtuellen Umgebung testen und optimieren können, bevor sie ihn in der realen Welt einsetzen. Es ist jedoch auch denkbar, dass die Simulation von Personen genutzt wird, welche sich für das Programmieren von Roboterarmen interessieren, aber keine formale Ausbildung in diesem Bereich haben. In diesem Fall könnte die Simulation als Lernplattform dienen, um die Grundlagen des Programmierens von Roboterarmen zu erlernen und zu experimentieren, ohne auf die physische Hardware angewiesen zu sein.

=== Technical Context

Die folgende Tabelle zeigt die zentralen technischen Schnittstellen des Digital Twins sowie deren jeweilige Aufgaben und Kommunikationsmechanismen.

[cols="2,3", options="header"]
|===
| Schnittstelle | Beschreibung

| Benutzer → System (Python API)
| Der Benutzer interagiert mit dem System über die bereitgestellte Python API. Diese stellt sämtliche Funktionen zur Steuerung des Roboterarms sowie zur Konfiguration der Simulationsumgebung bereit.
Über die API können sowohl Bewegungen (z. B. Gelenkwinkel) als auch Werkzeugfunktionen wie Greifer oder Pumpe gesteuert werden.

Zusätzlich ermöglicht die API die Erstellung und Verwaltung von Szenarien innerhalb der Simulation. Dazu gehören das Erzeugen, Löschen und Zurücksetzen von Objekten in der virtuellen Umgebung.
Es sind bereits mehrere vordefinierte Szenarien (Presets) vorhanden, welche einen schnellen Einstieg und Tests ermöglichen. Darüber hinaus können eigene Szenarien definiert werden, um spezifische Anwendungsfälle abzubilden.

| System (Python API) → Realer Roboter
| Die Python API ermöglicht die direkte Steuerung des realen MyPalletizer-Roboters.
Die Kommunikation erfolgt über eine serielle Schnittstelle (USB-A zu USB-C), über welche Steuerbefehle wie Gelenkbewegungen, Greiferaktionen oder Pumpensteuerung übertragen werden.

Je nach gewähltem Betriebsmodus können diese Befehle ausschliesslich an den realen Roboter gesendet oder parallel auch in der Simulation ausgeführt werden. Dadurch wird ein synchrones Verhalten zwischen realem System und digitalem Zwilling ermöglicht.

| System (Python API) → Unity-Simulation (Roboter)

UDP-Port: 5005
| Für die Simulation des Roboters erfolgt die Kommunikation zwischen der Python API und der Unity-Anwendung über UDP.
Dabei werden strukturierte Nachrichten (JSON-basiert) übertragen, welche Bewegungsbefehle sowie Zustandsänderungen des Roboters enthalten.

Die Unity-Simulation setzt diese Befehle in Echtzeit um und bildet das Verhalten des Roboters visuell nach. Im Betriebsmodus virtual erfolgt die Ausführung ausschliesslich in der Simulation, während im Modus both eine parallele Ausführung mit dem realen System stattfindet.

| System (Python API) → Unity-Simulationswelt

UDP-Port: 5006
| Die Verwaltung der Simulationsumgebung erfolgt ebenfalls über UDP-basierte Kommunikation mit Unity.
Dabei werden Befehle zum Erzeugen, Löschen oder Zurücksetzen von Objekten übertragen.

Die Objekte werden durch Parameter wie Position, Rotation, Skalierung und Farbe definiert und in der Unity-Szene instanziiert.
Zusätzlich können vordefinierte Szenarien geladen werden, welche mehrere Objekte in einer konsistenten Umgebung bereitstellen.

|===
