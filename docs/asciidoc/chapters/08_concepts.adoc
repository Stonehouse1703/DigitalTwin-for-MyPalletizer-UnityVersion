== Cross-cutting Concepts

In diesem Kapitel werden Konzepte beschrieben, die in mehreren Teilen des Systems verwendet werden.

=== Simulationskonzept

Die Simulation des Roboters wird in Unity umgesetzt. Dabei besteht der Roboter aus mehreren miteinander verbundenen Gelenken, sodass die Bewegung als kinematische Kette dargestellt werden kann.

[plantuml, target="diagram-kinematic-chain", format="png"]
----
@startuml
skinparam componentStyle rectangle

package "Kinematische Kette" {
    [Base] --> [J1]
    [J1] --> [J2]
    [J2] --> [J3]
    [J3] --> [J4]
    [J4] --> [End-Effektor]
}
@enduml
----

Die Trennung in einzelne Gelenke ermöglicht es, Bewegungsbefehle aus der Python-Anwendung direkt auf die simulierte Roboterstruktur abzubilden. Dadurch können Zielwinkel für die einzelnen Achsen unabhängig verarbeitet und in der Simulation dargestellt werden.

=== Moduskonzept

Das System unterstützt drei verschiedene Betriebsmodi:

* **virtual** – Ausführung nur in der Simulation
* **real** – Ausführung nur auf dem realen Roboter
* **both** – Ausführung gleichzeitig in Simulation und Hardware

Dieses Konzept ist zentral, da dieselbe Python-API in allen drei Fällen verwendet werden kann. Die Unterscheidung, wohin ein Befehl gesendet wird, erfolgt intern im System.

=== Konzept zur Trennung von Robot und World

Auf Python-Seite ist die Funktionalität bewusst in zwei Bereiche aufgeteilt:

* **Robot**
* **World**

`Robot` ist für Bewegungen, Werkzeuge und LED-Funktionen zuständig.
`World` verwaltet die Simulationsumgebung, also zum Beispiel das Erzeugen, Löschen und Zurücksetzen von Objekten.

Dies verbessert die Übersichtlichkeit und verhindert, dass sich Robotersteuerung und Weltverwaltung unnötig vermischen.

=== End-Effektor-Konzept

Der Roboter unterstützt unterschiedliche Werkzeuge, insbesondere Greifer und Pumpe. Diese Werkzeuge können sowohl über die Python-API ausgewählt als auch in der Simulation nachgebildet werden.

[plantuml, target="diagram-end-effector-concept", format="png"]
----
@startuml
[*] --> Werkzeug_waehlen
Werkzeug_waehlen --> Greifer : set_end_effector("gripper")
Werkzeug_waehlen --> Pumpe : set_end_effector("pump")

Greifer --> Greifer_Aktion : open / close
Pumpe --> Pumpe_Aktion : on / off
@enduml
----

Dieses Konzept sorgt dafür, dass verschiedene Werkzeugarten über dieselbe Schnittstelle angesprochen werden können.

=== Objekt- und Szenarienkonzept

Für die Simulationswelt werden einfache Objekte wie Boxen, Kugeln oder Zylinder verwendet. Diese können über die Python-Seite erstellt und in Unity dargestellt werden.

Zusätzlich unterstützt das System vordefinierte Szenarien (*Presets*), damit typische Testfälle schnell geladen werden können. Dadurch lassen sich wiederholbare Experimente und Demonstrationen einfacher durchführen.

Objekte werden dabei über ihren Namen angesprochen. So können sie später gezielt gelöscht oder verändert werden.

=== Fehlerbehandlung und Validierung

Ein weiteres wichtiges Konzept ist die Validierung von Eingaben vor der Ausführung.

Dazu gehören unter anderem:

* Begrenzung von Gelenkwinkeln
* Begrenzung der Geschwindigkeit
* Prüfung von Objektparametern
* Fehlerbehandlung bei ungültiger Konfiguration

Dadurch werden fehlerhafte Befehle möglichst früh erkannt und unzulässige Zustände vermieden.

=== Bereitstellungskonzept

Die Software wird über eine CI/CD-Pipeline automatisiert gebaut. Für die unterstützten Betriebssysteme werden Artefakte bereitgestellt, sodass Benutzer die passende Version direkt herunterladen und starten können.

Zusätzlich bleibt das Unity-Projekt zugänglich, damit die Simulation bei Bedarf weiterentwickelt oder angepasst werden kann.

=== Testkonzept

Die Qualität des Systems wurde vor allem durch funktionale Tests überprüft. Dazu gehören insbesondere:

* Testen der Kommunikation zwischen Python und Unity
* Testen der verschiedenen Betriebsmodi (`virtual`, `real`, `both`)
* Testen der Roboterbewegungen und Werkzeugfunktionen
* Testen der Weltbefehle wie `spawn`, `delete` und `clear`

Zusätzlich wurde die Simulation visuell überprüft, um sicherzustellen, dass Bewegungen und Objektinteraktionen plausibel dargestellt werden.
