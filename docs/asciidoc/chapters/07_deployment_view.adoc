== Deployment View

Die Verteilungssicht beschreibt die physische Infrastruktur und die Softwareumgebung, in der das System betrieben wird.

=== Infrastructure Level 1

Das System ist für den Betrieb auf einer Standard-Workstation ausgelegt. Die optionale Hardware wird lokal per USB angebunden.

[plantuml, target="diagram-deployment-l1", format="png"]
----
@startuml
node "Host PC (Windows/Linux/macOS)" as PC {
    component "Python Application"
    component "Unity Application"
}

node "MyPalletizer 260 M5" as Robot {
}

PC -right-> Robot : USB (Serial)
"Python Application" <-down-> "Unity Application" : UDP (Localhost)
@enduml
----

=== Infrastructure Level 2

Auf Ebene 2 betrachten wir die spezifischen Anforderungen und Komponenten der Betriebsumgebung.

==== Host-PC (Workstation)

Der Host-PC benötigt folgende installierte Softwarekomponenten:

* **Python Runtime (v3.12)**: 
    - Erforderlich für die Ausführung der Steuerlogik.
    - Beinhaltet Abhängigkeiten wie `pymycobot`, `numpy` und `pyserial`.
* **Unity-Standalone-Build**: 
    - Die vorkompilierte Version der Simulation. Diese enthält alle Assets und die C#-Receiver-Logik.
* **USB-Treiber (CP210x)**: 
    - Erforderlich für die Kommunikation mit dem M5Stack-Controller des Roboters über den virtuellen COM-Port.

==== Netzwerk-Infrastruktur

Obwohl das System primär auf `localhost` (127.0.0.1) betrieben wird, ermöglicht die UDP-basierte Architektur eine Verteilung der Komponenten im lokalen Netzwerk (LAN/WLAN):

[cols="1,2", options="header"]
|===
| Element | Beschreibung
| **Port 5005 (Roboter)** | Eingehender UDP-Verkehr für Bewegungsdaten. Muss in der Firewall des Host-PCs freigegeben sein.
| **Port 5006 (Welt)** | Eingehender UDP-Verkehr für Umgebungsbefehle.
| **Loopback Adapter** | Minimale Latenz (< 1ms) bei lokalem Betrieb.
|===

==== MyPalletizer 260 M5 (Hardware)

* **Firmware**: Auf dem Roboter muss die aktuelle Version von *atomMain* oder *Transponder* installiert sein, um die seriellen Befehle korrekt zu verarbeiten.
* **Energieversorgung**: Ein externes Netzteil (8.4V / 5A) ist für den Betrieb der Servos zwingend erforderlich, da die Stromversorgung über USB nicht ausreicht.
