== Building Block View

Die Bausteinsicht beschreibt die statische Struktur des Systems durch eine hierarchische Zerlegung in funktionale Komponenten.

=== Whitebox Overall System (Level 1)

Auf der obersten Ebene besteht das System aus zwei Hauptcontainern: der Python Control Layer und der Unity Visualization Layer.

[plantuml, target="diagram-building-blocks-l1", format="png"]
----
@startuml
!include <c4/C4_Component.puml>

System_Boundary(digital_twin_system, "Digital Twin System") {
    Container(python_api, "Python Control Layer", "Python 3.12", "Verantwortlich für Steuerlogik, Hardware-Abstraktion und API-Bereitstellung.")
    Container(unity_sim, "Unity Visualization Layer", "Unity 6 (C#)", "Verantwortlich für 3D-Rendering, Physik-Simulation und Umgebungsmanagement.")
}

Rel(python_api, unity_sim, "Steuerbefehle (Gelenkwinkel, Welt-Events)", "UDP / JSON")
@enduml
----

==== Contained Building Blocks

[cols="1,3", options="header"]
|===
| Baustein | Beschreibung
| **Python Control Layer** | Diese Schicht bildet das "Gehirn" des Systems. Sie stellt die Schnittstelle zum Benutzer bereit und entscheidet über die Weiterleitung der Befehle an Hardware oder Simulation.
| **Unity Visualization Layer** | Diese Schicht fungiert als "Körper". Sie empfängt Datenpakete und setzt diese in physikalische Bewegungen oder visuelle Änderungen in der 3D-Szene um.
| **Schnittstelle: UDP/JSON** | Ein asynchrones Nachrichtenprotokoll zur entkoppelten Kommunikation zwischen beiden Schichten.
|===

=== Level 2

In dieser Ebene werden die internen Strukturen der beiden Hauptcontainer detailliert.

==== Whitebox Python Control Layer

[plantuml, target="diagram-python-l2", format="png"]
----
@startuml
component "MyPalletizerController" as MPC
component "World" as W
component "Protocol" as P
component "pymycobot Library" as PMC

MPC -> P : nutzt für Nachrichtenaufbau
MPC -> PMC : steuert reale Hardware
W -> P : nutzt für Welt-Nachrichten
@enduml
----

* **MyPalletizerController**: Zentrale Klasse zur Orchestrierung. Implementiert das Strategie-Pattern für die Betriebsmodi.
* **World**: Verwalter der Simulationsumgebung. Kapselt Befehle zum Erzeugen und Löschen von Objekten.
* **Protocol**: Hilfsmodul zur zustandslosen Serialisierung von Befehlen in das JSON-Format (v1).

==== Whitebox Unity Visualization Layer

[plantuml, target="diagram-unity-l2", format="png"]
----
@startuml
component "UnityUdpBridge" as UUB
component "WorldUdpBridge" as WUB
component "MyPalletizerAdapter" as MPA
component "WorldSpawnManager" as WSM

UUB -> MPA : leitet Roboterbefehle weiter
WUB -> WSM : leitet Welt-Events weiter
@enduml
----

* **UnityUdpBridge**: Multithreaded Empfangseinheit für Roboter-relevante Daten auf Port 5005.
* **WorldUdpBridge**: Empfangseinheit für Welt-Manipulationen auf Port 5006.
* **MyPalletizerAdapter**: High-Level Controller, der die physikalische Gelenkkette koordiniert.
* **WorldSpawnManager**: Verarbeitet die Instanziierung von GameObjects basierend auf den empfangenen JSON-Daten.

=== Level 3

Auf Level 3 betrachten wir die feinmodularen Komponenten innerhalb der Unity-Simulation, die für die physikalische Präzision verantwortlich sind.

==== Whitebox MyPalletizerAdapter & Gelenke

Innerhalb des `MyPalletizerAdapter` werden vier `ArticulationJointActuator`-Komponenten gesteuert, die jeweils ein physikalisches Gelenk repräsentieren.

[plantuml, target="diagram-joints-l3", format="png"]
----
@startuml
class MyPalletizerArticulationAdapter {
    + MoveJointsAsync()
    + MoveJointsSync()
}

class ArticulationJointActuator {
    - ArticulationBody _ab
    + MoveTo()
}

MyPalletizerArticulationAdapter "1" *-- "4" ArticulationJointActuator
@enduml
----

* **ArticulationJointActuator**: Diese Komponente ist das Herzstück der Bewegungssimulation. Sie greift direkt auf Unitys `ArticulationBody` zu und nutzt Coroutinen, um eine geschwindigkeitsbasierte Bewegung (`degPerSec`) zu simulieren. Dies stellt sicher, dass der Roboterarm nicht "teleportiert", sondern physikalisch plausibel schwenkt.
* **ArticulationBody (Unity-intern)**: Verwendet den *Reduced Coordinate Space* zur stabilen Simulation der kinematischen Kette ohne Drifts.
